# Contributing to VoteVerse

Thank you for your interest in contributing to **VoteVerse**! We welcome all developers, designers, and civic enthusiasts.

## Code Standards
- **TypeScript**: All new code must be type-safe. Avoid `any`.
- **Testing**: Every new feature requires a corresponding Vitest or Node.js native test.
- **Accessibility**: Ensure all components meet WCAG 2.1 AA standards.
- **Security**: Never commit secrets or API keys. Use `.env`.

## Pull Request Process
1. Fork the repo.
2. Create a feature branch.
3. Ensure CI passes locally (`npm test`).
4. Submit your PR with a detailed description of changes.
